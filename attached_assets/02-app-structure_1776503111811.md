# Suprans App — Page Structure

Every page in the app, section by section. This is the wireframe reference. Four main tabs. Each tab has multiple screens within it. Copy for each screen is in `03-app-copy.md`.

---

## Global structure

### Bottom tab bar (always visible)

Height: 72px (larger than standard iOS 49px — intentional for big-button feel)
Background: white with subtle top border (1px, warm gray)
Four tabs, equally spaced, each with:
- 28px 3D icon (see icon generation guide)
- 11px label below icon
- Active state: icon in brand red, label in brand red, subtle pill-shaped background behind the icon
- Inactive state: icon and label in muted warm gray

The four tabs: Chat · Explore · Projects · Account

### Top status bar (varies per screen)

Height: 56px (standard native height)
Background: white on most screens, brand red on Chat screens when conversation is active
Contents vary per screen — back button, screen title, contextual action (search, filter, etc.)

### Safe area handling

All content respects iOS notch and Android status bar. Bottom tab bar accounts for home indicator on iOS and Android nav bar.

## Tab 1 — Chat

The most-used tab. Opens by default when the app is first launched each day.

### Screen 1.1 — Conversations list

**Top bar:**
- Logo + "Suprans" text on left
- Search icon on right (taps to open universal search)
- Status indicator (green dot = ops team online, gray = team offline, bell = new announcement)

**Body:**

Section 1: Pinned announcements (optional, only shows when admin has posted)
- Full-width card with warm cream background
- "New from Suprans" eyebrow label
- Short title + one-line description
- Dismissible (X in top-right)

Section 2: Active conversations
- Heading: "Your team"
- List of active chat threads, sorted by most recent message
- Each thread row (88px tall):
  - Left: 48px circular avatar (ops manager or group avatar)
  - Middle: Thread name (bold, 16px) + last message preview (14px, muted, truncated)
  - Right: Timestamp (12px muted) + unread count badge (if any, brand red circle with white number)
- Unread threads: left 3px brand red indicator strip, slightly bolder text

Section 3: All conversations
- Heading: "All conversations"
- Expandable list of older/resolved threads
- Same row format as active

**Floating action button (bottom right, above tab bar):**
- 56px circle, brand red, white plus icon
- Taps to start a new conversation
- Opens a modal: "What do you need help with?" with quick-start options

### Screen 1.2 — Individual conversation

**Top bar:**
- Back arrow on left
- Center: Thread name + "online" status below it (if applicable)
- Right: More menu (3-dot icon) — options: View participants, Project details, Mute, Archive

**Body (the chat view):**

Message list fills the screen. Scrolls vertically.

**Message bubble styles:**

For messages from client (the user):
- Aligned right
- Background: brand red
- Text: white
- Rounded bubble, pointed corner on bottom-right
- Timestamp below bubble in 11px muted

For messages from Suprans team (ops, AI, admin):
- Aligned left
- Background: warm cream (#F2ECE0)
- Text: near-black (#1A1612)
- Rounded bubble, pointed corner on bottom-left
- Avatar + name above the first message in a sequence
- Timestamp below last message of a sequence

For messages from AI assistant:
- Aligned left (same as team)
- Background: white with 1px warm gray border
- Small "AI" badge next to name
- Content: suggested response that user can tap to send or edit

For system messages:
- Full width, centered
- Small text, muted
- Examples: "Priya joined this chat", "Vendor HQ Dropshipping added to this conversation", "Sample shipped via DHL"

**Rich message types:**
- Document attachments: rectangular card with filename, size, preview thumbnail
- Image attachments: tap to expand full-screen
- Voice notes: waveform + duration + play button
- Product links: preview card with product image, name, price, tap to open product in in-app browser
- Payment requests: prominent card with amount, "Pay now" CTA, payment status
- Project milestones: visual chip showing stage completion

**Input bar (bottom, above tab bar):**

Height: 60px + safe area bottom padding
Background: white with 1px top border

Left to right:
- Plus icon (48px tap target): opens attachment menu (camera, gallery, document, voice, location)
- Text input (expandable, multi-line up to 4 lines visible)
- Voice record button (48px, circle, brand red when held)
- Send button (appears only when text is entered, 48px, brand red, white arrow icon)

**Quick replies (above input bar when AI has suggestions):**
- Horizontal scroll of 2-3 pill-shaped suggestion buttons
- Each pill: white background, 1px border, 14px text, brand red for primary action
- Tap = auto-fills input with suggestion text

### Screen 1.3 — New conversation modal

**Top bar:**
- Close X on left
- Title: "What do you need help with?"

**Body:**

Tile grid (2 columns) of common starting points:
- "Custom sourcing from China" (→ creates chinaimports RFQ thread)
- "Browse wholesale catalog" (→ opens chinaproducts webview)
- "Start dropshipping store" (→ creates USDrop thread)
- "Form a company / entity" (→ creates Legal Nations thread)
- "Franchise retail info" (→ creates EazyToSell thread)
- "Plan a China trip" (→ creates GoyoTours thread)
- "Something else" (→ opens general conversation with ops)

Each tile: 3D icon at top, title, 1-line description.

**Below the grid:**
"Prefer to explain yourself? [Describe what you need →]" — opens a text field for free-form requirement description, which AI classifies and routes.

The discovery surface. Shows everything Suprans offers. Clients browse and engage with new services.

**Top bar:**
- Title: "Explore Suprans"
- Right: Search icon

**Body:**

Section 1: Hero banner (dismissible carousel)
- Full-width 200px height

- Auto-rotates every 5 seconds (user can swipe manually)
- Each banner: large image/illustration, headline, CTA button
- Dot indicators at bottom of carousel

Section 2: Quick actions (horizontal scroll)
- 4-6 circular quick-action chips: "Start RFQ", "Browse catalog", "Form entity", "Track shipment", "Help center"
- Each chip: 56px circle with 3D icon, label below, 88px total width

Section 3: All services grid
- Heading: "Our services"
- 2-column grid of service tiles (each 160px tall)
- Service tiles:
  - USDrop AI — dropshipping automation
  - Legal Nations — entity formation
  - China Imports — custom sourcing
  - China Products — wholesale catalog
  - EazyToSell — franchise retail
  - GoyoTours — business travel
  - Keeraft — furniture import
  - La Bella Monte — watches & luxury goods
- Each tile: 3D icon top-left, service name, 1-line description, "Explore →" link at bottom

Section 4: Latest updates
- Heading: "Latest from Suprans"
- Vertical list of news/announcement cards
- Each card: thumbnail (left), headline, date, short description, tap to open full update

Section 5: FAQ / Help
- Single tile: "Questions? Get help →" linking to help center

### Screen 2.2 — Service detail

Each service tile opens a dedicated screen.

**Top bar:**
- Back arrow
- Service logo/name centered

**Body:**

Section 1: Hero card
- Full-width, 240px tall
- Background illustration or photo matching the service
- Title (serif feel, 28px)
- 2-line description
- Primary CTA: "Start a [service] conversation →" (opens new chat routed to that service's team)

Section 2: What we do
- 3-4 feature cards describing capabilities
- Each: icon, short title, 2-sentence description

Section 3: How it works
- Numbered steps (3-5 steps) explaining the process
- Each step: number badge, title, description

Section 4: Typical engagement
- Pricing model (if applicable)
- Timeline
- What's included

Section 5: Case studies / past work
- Horizontal scroll of 3-5 case study cards
- Each: thumbnail, project name, outcome

Section 6: CTA section
- Full-width callout with 2 actions:
  - "Start a conversation" (primary) → opens chat
  - "Visit [service].com" (ghost) → opens external web browser to service's full web presence

## Tab 3 — Projects

All active engagements a client has with Suprans.

### Screen 3.1 — Projects list

**Top bar:**
- Title: "Your projects"
- Right: Filter icon

**Body:**

Section 1: Active projects (with count)
- Vertical list of project cards
- Each card (120px tall):
  - Left: 48px service icon (matching the vertical)
  - Center: Project name, service vertical label, current stage status
  - Right: Stage progress dot + next milestone date
  - Below: Compact progress bar (4-6 dots representing stages)
- Tap a card to open project detail

Section 2: Completed projects (collapsible)
- Same card format but grayed out
- Shows completion date instead of next milestone

Section 3: Opportunities (new)
- Items the ops team has surfaced for the client's attention
- Each card: suggestion title, why it's relevant, CTA

### Screen 3.2 — Project detail

**Top bar:**
- Back arrow
- Center: Project name (truncated)
- Right: More menu (chat, share, archive)

**Body:**

Section 1: Status banner
- Full-width, 80px tall
- Current stage + next milestone + estimated completion date
- Background color reflects status: warm cream (in progress), green (on track), amber (attention needed), red (blocked)

Section 2: Progress timeline
- Vertical timeline (not horizontal on mobile — vertical is easier to read)
- 5-9 stages depending on project type
- Each stage: circle icon (filled if completed, outlined if current/future), stage name, description, completion date or estimate
- Current stage highlighted with brand red border

Section 3: Documents
- List of document cards per stage
- Each: document type icon, filename, size, upload date, download/view action

Section 4: Payment status
- Card showing total value, paid, pending, due dates
- If payment is due: prominent "Pay now" CTA

Section 5: Chat access
- Card with team avatars (ops + vendor if applicable)
- "Open conversation →" button that opens the related chat thread

Section 6: Project settings
- Notification preferences for this project
- Archive option

### Screen 3.3 — Project creation

For new projects started from a chat conversation or Explore screen.

**Top bar:**
- Close X
- Title: "New [service type] project"

**Body:**

Multi-step form (similar to chinaimports.in RFQ flow but mobile-optimized):
- Step 1: What are you sourcing / what do you need?
- Step 2: Timeline and logistics
- Step 3: Confirm your details

Each step is a full screen with:
- Progress dots at top
- Form fields (large, easy-tap)
- Large "Next" button at bottom (fixed position)

## Navigation patterns

### Deep linking

Every screen is deep-linkable. A URL like `suprans://chat/thread-xyz` opens the app directly to that conversation. This enables:
- Push notifications that open the exact screen
- Email/SMS links that route correctly
- Cross-promotion between Suprans web properties and the app

### Back navigation

- Always back arrow in the top-left when not on a tab root
- Swipe-from-edge gesture on iOS
- System back button on Android
- Clear back stack — no endless "previous screen" confusion

### Universal search (global)

Tap the search icon (top-right on most screens):
- Search across: conversations, projects, invoices, contacts, messages
- Results grouped by category
- Recent searches shown when search is empty
- Voice search supported

## Screen loading and empty states

### Loading

- Skeleton loaders (not spinners) for list screens
- Progress indicators for long operations (file upload, payment processing)
- Optimistic UI — show sent message immediately, retry in background if network fails

### Empty states

Every list screen has a designed empty state:
- 3D illustration (generated via nano banana, matching brand palette)
- Clear explanatory headline
- Description of what will appear here
- Primary CTA to take action that populates the state

Examples:
- No conversations yet: "Your first chat will appear here. [Start a conversation →]"
- No projects yet: "Your first project will show up here. [Explore services →]"
- No invoices yet: "Invoices will appear after your first order."

## Modal vs full-screen decision rule

Use modals (bottom sheets) for:
- Short confirmations (Delete this? Archive?)
- Quick filters
- Share actions
- Attachment pickers
- Payment flows that last less than 30 seconds

Use full screens for:
- Multi-step forms (project creation, RFQ submission)
- Reading long content (project details, invoices)
- Any action taking longer than 30 seconds

Modals should slide up from the bottom, not fade in from center. Native iOS behavior. On Android, similar bottom-sheet pattern.
