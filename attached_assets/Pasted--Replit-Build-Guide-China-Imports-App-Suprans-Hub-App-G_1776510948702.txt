# Replit Build Guide — China Imports App + Suprans Hub App

> **Goal of this document:** Give the Replit AI agent a clear, non-technical plan to rebuild our mobile app project as **two separate apps with two separate identities**, using the design system and codebase we already have in the Chinaimports-app repo as the foundation.
>
> **Repo reference:** `github.com/lakshaytakkar/Chinaimports-app` (current state has Flock + a mobile app currently named "Suprans" which is actually a sourcing app).

---

## 1. The Big Picture — What We Are Doing and Why

Right now, the mobile app in the current repo is labeled "Suprans" but it behaves like a sourcing app for Indian businesses importing from China. This is a branding mismatch. Suprans is our parent company — it is **not** a sourcing product. Suprans owns many businesses, and sourcing is just one of them.

We need to split the app into two clearly separated products:

### App 1 — **China Imports** (the sourcing app)
- This is what the current "Suprans" mobile app inside the repo actually is.
- We are **renaming and rebranding** it end-to-end as "China Imports."
- It stays in the current repo (Chinaimports-app) and keeps its 4-tab structure (Chat, Explore, Projects, Account).
- **No mention of Suprans anywhere inside this app.** It stands on its own as an independent sourcing product.
- Tagline: *"All-in-one one-stop solution for all your sourcing needs from China."*

### App 2 — **Suprans Hub** (the directory / parent app)
- This is a **brand new app** that we will build by duplicating the current codebase into a new Replit project and new GitHub repo.
- It uses the same tech stack, same design system, same mobile frame — but the purpose is completely different.
- Suprans Hub is a **directory app** that links out to all the products Suprans owns (Legal Nations, USDrop, China Imports, China Products, EazyToSell, Keeraft, GoyoTours, Events, etc.).
- Users do not source products inside Suprans Hub. They browse what Suprans offers, tap a tile, and either get redirected to that product's own landing page or opened into its own app/web experience.
- Suprans Hub is also where we centralize billing across all Suprans products and show upcoming events/tours.

### Why this split matters

Currently the single app is confusing: it talks about Suprans while actually doing sourcing. By splitting:
- **China Imports** becomes a focused, standalone product that can rank on its own, market itself, and not be diluted by unrelated things like company formation or franchises.
- **Suprans Hub** becomes the umbrella where existing clients see the full ecosystem, discover new services, manage billing in one place, and find upcoming events.
- Design system, components, layout, and code are shared — we are not starting from zero on either side.

### 2.4 The Chat tab — this is the most important part

The Chat tab is not just a feature — it is the **core reason this app exists**. It solves a specific operational problem: today, every import client ends up in a separate WhatsApp group with the sourcing partner and the freight forwarder. This creates dozens of parallel chats, no admin visibility, no audit trail, and impossible follow-up at scale.

The Chat tab inside China Imports **replaces that WhatsApp group workflow**:
- Every sourcing request (RFQ) the client submits gets its own chat thread inside the app.
- Client messages, ops team messages, buying agent messages, and file attachments (quotes, invoices, QC reports, shipping docs) all live inside one thread, attached to one RFQ.
- Our admin side has full visibility — every conversation across every client is readable, searchable, and assignable from a single ops dashboard.
- WhatsApp stays only as a nudge channel ("new update — open app"), not as the conversation.

Make sure Replit understands: the Chat tab is not a generic chat — it is a client ↔ ops team conversation space **scoped per sourcing order/RFQ**, with full admin visibility on our side.

### 2.5 Tabs — what each one does inside China Imports

| Tab | Purpose |
|---|---|
| **Chat** | Per-RFQ conversation threads. Client ↔ ops ↔ buying agent. File attachments, quote cards, status updates. The system of record for every sourcing conversation. |
| **Explore** | "Mini China" — city-wise product discovery. Foshan (furniture, ceramics), Yiwu (toys, small goods, seasonal), Guangzhou (gifts, accessories), Shenzhen (electronics, watches), Dongguan (hardware, outdoor). Tap a city → see product categories → submit RFQ from the category. |
| **Projects** | Active and past sourcing orders. Each order shows its 9-stage progress bar (RFQ received → Sourcing → Quote → Sample → Approval → Bulk → QC → In Transit → Delivered). Documents downloadable per order. |
| **Account** | Client profile, GST, company details, documents, payment history, notifications, logout. |

## 3. App 2 — Build a New "Suprans Hub" App by Duplicating the Current Codebase

### 3.1 How to create it
1. Duplicate the entire current repo (Chinaimports-app) into a new Replit project.
2. Push it to a new GitHub repo named something like `suprans-hub-app`.
3. Keep the Flock app in the new repo too if desired (or strip it — your choice, not critical).
4. The **China Imports app stays only in the original repo** (after the rename from section 2).
5. In the new duplicated repo, we now transform the routes, tabs, and content into the Suprans Hub directory app described below.

### 3.2 What Suprans Hub actually is
Suprans Hub is a **mobile directory app** for all Suprans ecosystem products. Think of it like a mini app store for our own services. A user (existing client, new prospect, or team member) logs in, sees everything Suprans offers, taps what interests them, and either:
- Gets redirected to that product's dedicated website/app, OR
- Sees the details, pricing, and CTAs for that product inside Suprans Hub itself.

### 3.3 Tab structure for Suprans Hub (replaces the 4 tabs from the source app)

| Tab | Purpose |
|---|---|
| **Home** | Directory grid of all Suprans products (tiles). This is where users spend 80% of their time. Each tile opens a product detail page with the product's pitch, features, pricing snippet, and CTA. |
| **Events** | Upcoming events, webinars, China sourcing tours, workshops, product launches. A calendar view + upcoming event cards. |
| **My Services** | What the logged-in user is already using or subscribed to across Suprans products. Their active LLC formation, their USDrop subscription, their active sourcing orders in China Imports, their franchise store, etc. One unified "what am I paying for across Suprans" view. |
| **Account** | Profile, billing history across all Suprans products, saved payment methods, documents, notifications, support, logout. |

### 3.4 Onboarding for Suprans Hub
Keep the same 3-slide carousel → phone OTP structure. New copy:
- **Slide 1:** *"One Suprans account. Every Suprans product."*
- **Slide 2:** *"From US company formation to China sourcing — all your growth tools in one place."*
- **Slide 3:** *"Sign in once. Manage everything."*

## 8. Content Rules for Every Tile and Page

- **Primary name is always the trading brand** (e.g., "USDrop AI"), never the internal description.
- **Subtitle immediately explains what the name stands for** (e.g., "USA Dropshipping Enablement Programme") so a first-time visitor instantly knows what it is.
- **One-liner is a confident, specific, benefit-led sentence** — not marketing fluff. It says what the user gets, not what we do.
- **No emojis** in any of the copy or inside the app UI (per our brand standard).
- **Professional tone throughout** — no AI-generated feel, no excessive exclamation, no Gen-Z slang.
- **All redirects open in the same tab for mobile web, or external browser for native.** Never break the user's session flow unexpectedly.
